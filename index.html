        Stratigraphy

        A layered network visualization tool

        About

        Stratigraphy renders multi-year relational datasets as stacked network layers in 3D space. Each horizontal plane represents a year, with nodes clustered by category. Connections within a year appear as solid lines; connections across years appear as dashed lines. The size of each circle reflects the volume of activity in that year.

        Named after the geological study of rock layers, Stratigraphy reveals how relationships, projects, and structures evolve over time — letting you see both the big picture and the fine detail.

        How to use

        • Orbit — Click and drag to rotate the view
        • Zoom — Scroll to zoom in and out
        • Hover — Mouse over a node to see its tooltip and highlight its connections
        • Click — Click a node to zoom in, open the detail panel, and explore its connections
        • Filter — Use the sidebar to filter by year, category, or country. Use "clear" to deselect all, then pick specific items
        • Search — Type a name, project code, city, or country to find matching nodes

        Data format

        Stratigraphy reads a single data.json file with the following structure:

        {
  "meta": {
    "title": "...",
    "description": "..."
  },
  "nodes": [
    {
      "id": "unique-id",
      "label": "Display Name",
      "year": 2024,
      "project": "Category",
      "title": "Full Title",
      "city": "City",
      "country": "Country",
      "website": "https://..."
    }
  ],
  "edges": [
    {
      "source": "node-id-1",
      "target": "node-id-2",
      "type": "intra | inter",
      "weight": 3
    }
  ]
}

        nodes — Each node must have an id, label, year, and project (used for clustering). All other fields are optional metadata shown in the detail panel.

        edges — Each edge connects two node IDs. Set type to "intra" for same-year connections (solid lines) or "inter" for cross-year connections (dashed lines). weight is optional (1–5).
